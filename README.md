# C++ 학습 정리

## 빌드 시스템

- `g++` 직접 사용은 파일이 적을 때 적합
- 파일이 많아지면 빌드 시스템 사용
  - **Make**: Makefile 작성, 변경된 파일만 재컴파일
  - **CMake**: CMakeLists.txt → Makefile 생성 → make → g++ → 실행파일
- `.vscode/tasks.json`: VSCode 전용 빌드 단축키 설정 (실제 빌드와 무관)

## 자료형

- `short`, `int`, `long`, `long long`: 정수형
- `float`, `double`: 부동소수점 (실수형)
- `unsigned`: 음수 없이 양수 범위를 두 배로 확장
- `auto`: 컴파일러가 타입을 자동 추론 (컴파일 타임에 확정, 이후 변경 불가)
- `climits`: 각 자료형의 최솟값/최댓값 상수 제공

## 포인터

- `int* p`: 주소값을 저장하는 포인터 변수
- `&a`: 변수 a의 주소
- `*p`: 포인터가 가리키는 주소의 값 (역참조)
- 배열 이름은 첫 번째 요소의 주소 (`arr == &arr[0]`)

### 복사 전달 vs 참조 전달
| 방식 | 예시 | 특징 |
|------|------|------|
| 복사 | `int b = a` | 독립적인 복사본 |
| 참조(포인터) | `int* b = &a` | 원본에 직접 접근 |
| 참조 | `int& b = a` | 원본의 별명 |

### 메모리 할당
| 방식 | 위치 | 해제 |
|------|------|------|
| `int a` | 스택 | 자동 |
| `new int` | 힙 | `delete` 직접 해제 |
| `std::unique_ptr` | 힙 | 자동 (스마트 포인터) |

### C/C++ 메모리 관리 발전
```
C: malloc/free → C++ 초기: new/delete → 현대 C++: 스마트 포인터
```

## 참조 변수 (Reference)

- `int& ref = a`: 변수 `a`의 **별명(alias)** — 같은 메모리를 가리킴
- 선언과 동시에 반드시 초기화해야 함 (포인터와 달리 나중에 대상 변경 불가)
- `ref = 10` 하면 원본 `a`도 10으로 바뀜

```cpp
int a = 5;
int& ref = a;   // ref는 a의 별명
ref = 10;       // a == 10
```

### 포인터 vs 참조 변수 비교

| | 포인터 (`int* p`) | 참조 (`int& r`) |
|--|-------------------|-----------------|
| 초기화 | 나중에 가능 (`nullptr`) | 선언 시 반드시 초기화 |
| 재할당 | 가능 (`p = &b`) | 불가 (항상 같은 대상) |
| null 가능 | 가능 | 불가 |
| 역참조 문법 | `*p` 필요 | 그냥 변수처럼 사용 |
| 주 용도 | 동적 메모리, 배열 | 함수 매개변수, 별명 |

## 값 전달 / 포인터 전달 / 참조 전달

### 값 전달 (Pass by Value)

```cpp
void add(int x) { x += 10; }  // 복사본이 바뀜
int a = 5;
add(a);  // a는 여전히 5
```

- 인자의 **복사본**이 생성됨
- 원본 변경 불가
- 큰 객체면 복사 비용이 큼

### 포인터 전달 (Pass by Pointer)

```cpp
void add(int* x) { *x += 10; }  // 역참조로 원본 변경
int a = 5;
add(&a);  // a == 15
```

- 주소를 복사해서 전달 → **원본 직접 수정** 가능
- `nullptr` 전달 가능 → 호출 전 null 체크 필요
- C 스타일, 현대 C++에서는 참조 전달 선호

### 참조 전달 (Pass by Reference)

```cpp
void add(int& x) { x += 10; }  // 별명으로 원본 변경
int a = 5;
add(a);  // a == 15
```

- 복사 없이 원본에 직접 접근 (포인터 전달과 동일한 효과)
- 문법이 깔끔하고 `nullptr` 위험 없음
- 읽기 전용이면 `const int& x` 사용 권장

### 세 방식 비교 요약

| | 값 전달 | 포인터 전달 | 참조 전달 |
|--|---------|------------|----------|
| 원본 수정 | 불가 | 가능 | 가능 |
| 복사 비용 | 있음 | 없음 (주소만) | 없음 |
| null 가능 | - | 가능 | 불가 |
| 문법 | 간단 | `*`, `&` 필요 | 간단 |
| 권장 상황 | 작은 기본형 | C 스타일 / nullable | 객체, 큰 구조체 |

> **실무 기준**: 기본형은 값 전달, 객체는 `const&` (읽기) 또는 `&` (수정) 전달

## 구조체 vs 클래스

| | struct | class |
|--|--------|-------|
| 기본 접근제어 | public | private |
| 용도 | 단순 데이터 묶음 | 캡슐화 + 메서드 |
| 자바 비유 | DTO | class |

- 현업에서는 `class`를 훨씬 더 많이 사용
- 큰 객체 전달 시 `const &` 사용 권장 (복사 비용 절감 + 수정 불가)

## enum

```cpp
// 구식 (C 스타일)
enum Color { RED, GREEN, BLUE };

// 현대 C++
enum class Color { RED, GREEN, BLUE };
Color c = Color::RED;  // 스코프 연산자 :: 사용
```

| C++ | Java |
|-----|------|

| `std::sort()` | `Collections.sort()` |

## 헤더파일 (.h) 과 소스파일 (.cpp)

- `.h`: 함수 원형, 클래스 선언, 구조체 정의 (인터페이스 역할)
- `.cpp`: 실제 함수 구현 (정의)
- 자바 인터페이스와 구현 클래스 관계와 유사

```
math.h (원형)  →  main.cpp에서 #include "math.h"
math.cpp (정의)
```

## 인라인 함수

- 함수 호출 시 점프 없이 호출 위치에 코드를 직접 삽입
- 짧고 자주 호출되는 함수에만 효과적
- 현대 컴파일러는 알아서 인라인 처리 → 직접 쓸 일 적음

```cpp
inline int add(int a, int b) { return a + b; }
```

### 함수 원형 vs 정의 위치
| 상황 | 위치 |
|------|------|
| 원형 없이 | 정의를 main() 위에 |
| 원형 있으면 | 원형은 main() 위, 정의는 main() 아래 |

## C++ vs Java 비교

| 개념 | C++ | Java |
|------|-----|------|
| 기본 전달 방식 | 복사 | 참조 (객체) |
| 메모리 관리 | 직접 관리 | GC 자동 관리 |
| enum 스코프 | `Color::RED` | `Color.RED` |
| 범위 for문 | `for (auto& x : list)` | `for (int x : list)` |
| 배열 | `std::vector` | `ArrayList` |
